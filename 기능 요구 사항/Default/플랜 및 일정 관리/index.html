    플랜 및 일정 관리

    기능 요구 사항¶

    플랜 및 일정 관리(MVP)¶

    1️⃣ 술자리 플랜 생성 · 편집 · 공유¶

    ID 우선순위 내용
    FR1 MUST 사용자는 1차, 2차, 3차 술자리 장소를 사전에 계획하여 하나의 술자리 플랜(코스)을 생성할 수 있어야 한다.
    FR2 SHOULD 술자리 플랜에는 각 장소의 예상 도착 시간과 장소 간 이동 경로 정보가 포함되어야 한다.
    FR3 MUST 사용자는 생성한 술자리 플랜을 친구들과 공유할 수 있어야 하며, 친구들은 해당 플랜에 대해 투표(선호 표시)를 할 수 있어야 한다.
    FR4 MUST 사용자는 생성된 술자리 플랜을 언제든지 수정 또는 삭제할 수 있어야 한다.
    FR7 SHOULD 사용자는 술자리 플랜에 장소별 메모(예: 예약 정보, 입장 조건, 주의사항 등)를 남길 수 있어야 한다.
    FR8 SHOULD 사용자는 과거에 실행된 술자리 플랜을 기준으로 동일하거나 유사한 플랜을 빠르게 재사용하거나 복제할 수 있어야 한다.
    FR9 SHOULD 시스템은 사용자의 플랜 기록을 기반으로 자주 방문하는 루트(지역·코스)를 자동으로 추천하거나, 플랜 작성 시 자동완성 기능을 제공할 수 있어야 한다.

    2️⃣ 술자리 일정 · 알림 관리¶

    ID 우선순위 내용
    FR10 MUST 사용자는 특정 날짜 및 시간에 술자리 일정을 저장할 수 있어야 하며, 시스템은 해당 일정에 대해 알림 기능을 제공해야 한다.
    FR11 MUST 사용자는 자신의 과거 술자리 일정을 이력 형태로 조회할 수 있어야 한다.
    FR12 SHOULD 사용자는 술자리 일정에 대해 개별 또는 그룹 알림 시점(예: 30분 전, 하루 전, 당일 아침 등)을 설정·조정할 수 있어야 한다.

    3️⃣ 캘린더 · 시각화 · 공유 캘린더/연동¶

    ID 우선순위 내용
    FR13 MUST 시스템은 술자리 일정 정보를 캘린더 뷰 형태로 시각화하여 제공해야 하며, 사용자는 월/주 단위로 일정을 한눈에 확인할 수 있어야 한다.
    FR14 MUST 사용자는 각 일정에 대해 간단한 제목과 술자리 유형(예: 회식, 친구모임, 데이트 등)을 아이콘 형태로 표시하여 서로 다른 일정을 쉽게 구분할 수 있어야 한다.
    FR15 SHOULD 사용자는 캘린더에서 원하는 날짜를 클릭하여 새로운 술자리 약속을 간편하게 등록하거나, 기존 일정을 바로 편집할 수 있어야 한다.
    FR16 MUST 시스템은 사용자의 술자리 일정과 연동되는 주간/월간 캘린더 UI를 제공해야 하며, 중요 일정에는 색상, 배지 등 시각적 강조 표시를 지원해야 한다.
    FR17 MUST 사용자는 일정이 있는 날짜에 대해 간단한 요약 정보(예: “19:00 강남 1차 회식”)를 툴팁 또는 팝업 형태로 확인할 수 있어야 한다.
    FR18 SHOULD 시스템은 과거 술자리 일정을 캘린더에서 다시 클릭했을 때, 해당 일정의 상세 이력(장소, 참여자, 메모 등)을 회고용으로 열람할 수 있도록 해야 한다.
    FR19 COULD 사용자는 친구들과 공유 캘린더를 생성하여 서로의 술자리 약속을 확인하고, 이를 일정 조율(시간/장소 조정)에 활용할 수 있어야 한다.
    FR20 COULD 시스템은 사용자의 중요한 약속(예: 기념일, 생일)과 술자리 일정이 겹치지 않도록, 외부 캘린더(Google Calendar 등)와 연동할 수 있는 옵션을 제공할 수 있어야 한다.
